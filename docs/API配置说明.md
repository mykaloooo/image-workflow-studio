## 当前配置（官网 4K 版）

### system_config.json

```json
{
  "gemini_api_key": "AIzaSy...",              // Google 官方 API Key
  "gemini_api_url": "",                        // 留空 = 使用官方地址
  "gemini_model": "gemini-3-pro-image-preview",
  "proxy_url": "socks5://127.0.0.1:10808",    // v2rayN SOCKS5 代理
  ...
}
```

### 字段说明

| 字段 | 值 | 说明 |
|------|------|------|
| `gemini_api_key` | `AIzaSy...` | Google 官方 API Key（付费，支持 4K） |
| `gemini_api_url` | **留空** | SDK 自动使用 `https://generativelanguage.googleapis.com/` |
| `gemini_model` | `gemini-3-pro-image-preview` | 图片生成模型 |
| `proxy_url` | `socks5://127.0.0.1:10808` | v2rayN SOCKS5 代理（翻墙） |

---

## v2rayN 代理端口说明

| 端口 | 协议 | 格式 | 状态 |
|------|------|------|------|
| **10808** | SOCKS5 | `socks5://127.0.0.1:10808` | ✅ 默认开启 |
| **10809** | HTTP | `http://127.0.0.1:10809` | ⚠️ 需手动开启 |

**注意**：v2rayN 默认只开启 SOCKS5 代理 (10808)，所以配置中使用 `socks5://` 格式。

## 两种使用模式

### 模式 1：官网直连（当前，支持 4K）

```json
{
  "gemini_api_key": "AIzaSy... (Google 官方 Key)",
  "gemini_api_url": "",
  "proxy_url": "socks5://127.0.0.1:10808"
}
```

**特点**：
- ✅ 支持 4K 分辨率
- ✅ 使用 Google 官方 API
- ⚠️ 需要翻墙（v2rayN）
- ⚠️ 需要付费 API Key

### 模式 2：中转服务（免费，不支持 4K）

```json
{
  "gemini_api_key": "sk-xxx (中转服务 Key)",
  "gemini_api_url": "http://127.0.0.1:8045",
  "proxy_url": ""
}
```

**特点**：
- ❌ 可能不支持 4K
- ✅ 不需要翻墙
- ✅ 可能有免费额度

## 常见问题

### Q1: 报错 "10061 由于目标计算机积极拒绝"

**原因**：代理端口未开启或配置错误

**解决**：
1. 检查 v2rayN 是否运行
2. 使用 `socks5://127.0.0.1:10808` 而不是 `http://127.0.0.1:10809`

### Q2: 如何切换到免费中转模式？

在系统设置中修改：
- `gemini_api_url` 填入中转地址（如 `http://127.0.0.1:8045`）
- `proxy_url` 留空

### Q3: 生成的图片不是 4K？

检查是否使用了中转服务，中转服务可能不支持 4K。切换到官网直连模式。
