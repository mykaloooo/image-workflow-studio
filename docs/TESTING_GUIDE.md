# 图片标注功能测试指南

## 测试步骤

### 1. 启动项目

```bash
# 方式 1: 使用启动脚本
双击 启动.bat

# 方式 2: 手动启动
cd backend
python app.py
```

浏览器访问: http://localhost:5001

### 2. 准备测试图片

1. 在画布上创建一个图片生成节点
2. 输入提示词，例如: "一只可爱的猫咪"
3. 点击生成按钮，等待图片生成完成

### 3. 测试标注功能

#### 3.1 打开标注编辑器
1. 点击生成的图片节点
2. 在详情面板中找到 "✏️ 标注" 按钮
3. 点击按钮，标注编辑器应该以全屏模式打开

#### 3.2 测试画笔工具
1. 点击 "✏️" 画笔按钮
2. 在图片上拖动鼠标绘制
3. 调整颜色选择器和线条粗细滑块
4. 验证绘制效果

#### 3.3 测试箭头工具
1. 点击 "↗️" 箭头按钮
2. 在图片上点击起点，拖动到终点
3. 释放鼠标，应该生成带箭头的线条

#### 3.4 测试矩形工具
1. 点击 "▭" 矩形按钮
2. 在图片上拖动绘制矩形
3. 验证矩形是透明填充、彩色边框

#### 3.5 测试文字工具
1. 点击 "Tt" 文字按钮
2. 在图片上点击任意位置
3. 输入文字内容
4. 调整字体大小滑块
5. 双击文字验证可以再次编辑

#### 3.6 测试选择工具
1. 点击 "↖️" 选择按钮
2. 点击之前创建的标注对象
3. 拖动对象移动位置
4. 调整对象大小

#### 3.7 测试撤销/重做
1. 进行几次绘制操作
2. 点击 "↶" 撤销按钮
3. 验证操作被撤销
4. 点击 "↷" 重做按钮
5. 验证操作被恢复

#### 3.8 测试删除功能
1. 选择一个标注对象
2. 点击 "🗑️" 删除按钮
3. 验证对象被删除

#### 3.9 测试保存功能
1. 完成标注后，点击 "保存标注" 按钮
2. 等待上传完成
3. 验证弹出 "标注已保存" 提示
4. 关闭编辑器
5. 验证节点中的图片已更新为标注后的版本

#### 3.10 测试取消功能
1. 重新打开标注编辑器
2. 进行一些标注操作
3. 点击 "取消" 按钮
4. 验证编辑器关闭且图片未更新

### 4. 测试工作流集成

1. 在标注后的图片节点右侧拖出连线
2. 创建新的图片生成节点
3. 输入提示词，使用标注后的图片作为参考
4. 验证可以正常生成新图片

### 5. 测试项目保存

1. 完成标注后，保存项目
2. 重新加载项目
3. 验证标注后的图片正确显示

## 预期结果

✅ 所有工具都能正常工作
✅ 标注可以正确保存和显示
✅ 撤销/重做功能正常
✅ 标注后的图片可以用于工作流
✅ 项目保存/加载正常

## 常见问题

### Q: 标注编辑器打不开？
A: 检查浏览器控制台是否有错误，确保 fabric.js 正确加载

### Q: 保存标注失败？
A: 检查后端服务是否正常运行，查看后端日志

### Q: 图片显示不正常？
A: 检查图片 URL 是否正确，确保后端 outputs 目录有读写权限

### Q: 标注工具不响应？
A: 刷新页面重试，检查浏览器兼容性（推荐 Chrome/Edge）
